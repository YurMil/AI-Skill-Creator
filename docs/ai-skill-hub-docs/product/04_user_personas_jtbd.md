# User Personas and Jobs To Be Done

## Persona 1: AI application developer

### Profile

Builds internal tools, coding agents, research agents, document processors, and automation workflows.

### Goals

- Find reusable skills quickly.
- Understand compatibility with selected agent runtime.
- Download and customize a package.
- Avoid unsafe third-party instructions.

### Jobs to be done

- When I build an agent, I want to search for skills by task so that I can avoid writing everything from scratch.
- When I download a skill, I want to see its files and risk level so that I can trust what I install.

## Persona 2: Skill author

### Profile

Creates reusable skill packages from internal knowledge, scripts, prompts, and workflows.

### Goals

- Publish high-quality skills.
- Receive validation feedback.
- Manage versions and changelogs.
- Build reputation.

### Jobs to be done

- When I create a skill, I want templates and validation so that my package follows the expected standard.
- When I publish a new version, I want a changelog and review workflow so that users understand what changed.

## Persona 3: Prompt engineer / automation specialist

### Profile

Turns repeatable workflows into AI-assisted processes for teams.

### Goals

- Convert SOPs and runbooks into skills.
- Compose multiple skills into packs.
- Provide examples and output templates.

### Jobs to be done

- When I have a process document, I want to generate a skill draft so that I can quickly operationalize it.
- When a workflow needs multiple capabilities, I want to visually combine skills into a pack.

## Persona 4: Security reviewer

### Profile

Responsible for reviewing skills before internal or public use.

### Goals

- Detect risky instructions.
- Detect destructive scripts.
- Prevent data exfiltration.
- Enforce organization policies.

### Jobs to be done

- When someone submits a skill, I want a structured risk report so that I can approve or reject it consistently.
- When a skill changes, I want to compare versions so that I can focus review effort on the diff.

## Persona 5: Organization admin

### Profile

Manages AI platform usage across a company or team.

### Goals

- Maintain a private skill registry.
- Control who can publish and approve.
- Enforce trusted sources.
- Keep audit records.

### Jobs to be done

- When teams use agent skills, I want an approved internal catalog so that employees use safe and standardized packages.
- When an incident occurs, I want audit logs so that I can understand what was published, downloaded, and approved.

## Persona 6: Learner / beginner

### Profile

New to LLM agent skills and wants guidance.

### Goals

- Understand what skills are.
- Learn how to write `SKILL.md`.
- Use examples and templates.

### Jobs to be done

- When I start, I want a guided tutorial so that I can create my first valid skill.
- When validation fails, I want explanations and links to documentation.
