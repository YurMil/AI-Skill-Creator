# Roadmap

## Phase 0: Discovery and planning

### Duration

1-2 weeks.

### Deliverables

- Product vision.
- PRD.
- Technical specification.
- Threat model.
- MVP backlog.
- UX wireframes.
- Initial architecture.

### Exit criteria

- MVP scope approved.
- Architecture baseline approved.
- Security assumptions documented.
- Development tasks estimated.

## Phase 1: MVP

### Duration

4-8 weeks.

### Deliverables

- Web catalog.
- User accounts.
- Skill editor.
- ZIP upload and export.
- Basic validation.
- Basic review workflow.
- Knowledge base.
- Admin review panel.

### Exit criteria

- End-to-end flow works: create, validate, submit, approve, publish, download.
- P0 test suite passes.
- MVP documentation is published.

## Phase 2: Beta

### Duration

8-12 weeks.

### Deliverables

- Semantic search.
- GitHub import.
- Version comparison.
- Security scanner v1.
- Collections.
- Ratings and reviews.
- Eval case support.
- Improved visual canvas.

### Exit criteria

- At least 100 catalog skills.
- Search quality reviewed with test queries.
- Review workflow handles real submissions.
- Security scanner blocks known malicious fixtures.

## Phase 3: Production

### Duration

12-20 weeks.

### Deliverables

- Organization workspaces.
- Private registries.
- RBAC.
- Audit logs.
- Sandbox validation.
- API keys.
- Public API.
- Observability dashboards.
- Backup and recovery process.

### Exit criteria

- Production SLOs defined and monitored.
- Security review completed.
- Incident response process documented.
- Data backup and restore tested.

## Phase 4: Enterprise platform

### Duration

20+ weeks.

### Deliverables

- SSO/SAML.
- Policy engine.
- Approval workflows.
- Signed packages.
- CI/CD integrations.
- Organization-level compliance reports.
- Advanced sandbox and runtime compatibility checks.

### Exit criteria

- Enterprise pilot completed.
- Organization governance model validated.
- Large private registry performance tested.
